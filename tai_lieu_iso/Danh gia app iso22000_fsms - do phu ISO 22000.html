Ngày: 14/06/2026 · Người đánh giá: Ban ISO (hỗ trợ AI) · Nguồn: README + chỉ mục thiết kế (docs/design) trên GitHub. Đánh giá ở mức kiến trúc/độ phủ; cần xác minh thêm ở mức code (clone repo).

1. Nhận định nhanh (TL;DR)

App được thiết kế bài bản và rộng: 18 module phủ gần như toàn bộ điều khoản ISO 22000:2018 (tài liệu, đánh giá nội bộ, NCR/CAR, thu hồi, khẩn cấp, thẩm tra, bối cảnh/rủi ro, thiết bị, NCC, sản xuất, PRP, lưu mẫu, HACCP, xem xét lãnh đạo, đào tạo, truyền thông, truy xuất). Đây không phải hệ thống "thiếu số hóa" về mặt khung quản lý — phần khung rất tốt.

Cảm giác "chưa đủ" của anh có cơ sở, nhưng nằm ở phần đặc thù an toàn thực phẩm FMCG (allergen, PRP đầy đủ theo ISO/TS 22002-1, giám sát môi trường, kiểm soát phụ gia, nhãn) và ở chiều sâu vận hành/dữ liệu — chứ không phải ở khung ISO. Mức độ phức tạp: Cao.

4. Rủi ro kỹ thuật / nợ tài liệu (cần làm sạch)

  • Số liệu README chưa nhất quán: phần 1 ghi "123 DocType" nhưng blueprint ghi "47 doctype + 28 child"; fixtures ghi 47 / 205 / 255 records ở các chỗ khác nhau. Cần đối chiếu lại thực tế cài đặt.
  • Độ phủ test thấp: chỉ 5 module test cho hệ thống rất lớn — rủi ro hồi quy khi nâng cấp.
  • 48 "Custom Permission read-only" trên DocType core ERPNext: cần rà để tránh xung đột khi nâng cấp ERPNext.
  • Cần kiểm chứng blueprint ↔ implementation: app có thật sự install/migrate sạch và DocType khớp thiết kế không (chạy bench migrate + run-tests trên staging).
  • Cần xác minh ở mức code (chưa đọc được do GitHub render JS + chưa clone được): hooks.py (auto-create chain), workflow_validation.py, các whitelisted method (kiểm tra có frappe.has_permission), fixtures có đủ Custom Field/Role/Workflow không.

5. Kết luận & bước tiếp theo

App đã phủ khung quản lý ISO 22000 ở mức tốt. Để "đủ" cho một hệ thống ATTP FMCG vận hành thực tế (và sẵn sàng FSSC 22000/xuất khẩu), cần bổ sung 4–5 mảng đặc thù: allergen, PRP đầy đủ, giám sát môi trường, kiểm soát phụ gia/nhãn, food defense/fraud — đúng các tài liệu em vừa soạn cho anh (QT 09 allergen, QT 10 dị vật/hóa chất/rework, KH.KN.01 kiểm nghiệm/EMP, bảng định mức phụ gia) → có thể đưa thẳng thành DocType.

Đề xuất: (1) chạy X-ray sâu ở mức code (clone repo) để xác minh các điểm mục 3H/4; (2) lập backlog build các module còn thiếu và chuyển sang quy trình build có kiểm soát.
